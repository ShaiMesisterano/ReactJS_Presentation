React

  • A Javascript library for building user interfaces
  • Not a Full MVC Library
  • The V in MVC

Vitual DOM

  • Browser's DOM is expensive
  • For every "re-render", React creates a virtual DOM
    1. React compares the virual DOM to the current DOM
    2. The differences are then processed as native javascript manipulations
    3. React will then make the minimal number of DOM changes to the actual DOM

The component is the new king

  • No more seperation between: UI, logic, data (MVC)
  • Each component manages its scope
  • Microservices-like

Application Initiation

  • React.RenderComponent(<root component name>);
    1. Create the component
    2. Call "render()"
    3. For each sub-components:
      • GoTo Step#1
      • On "state" change:
        • GoTo Step#2

Advantages

  • Fast Client-side rendering
  • Minimal DOM manipulation
  • Plays well with others (Backbone)

React Components

  • Defined by markup & sub-components
  • Writes into React.DOM (instead of actual DOM)
  • React.DOM represents the virtual DOM
  • React uses "data-" attribute: "data-reactid"
  • "data-reactid" enables smart diff between DOM states

JSX

JSX

  • HTML replacement
  • JS+XML
  • JS syntax extension
  • Enables XML snippets inside JS code

JSX vs. HTML

  • Final slash is mandatory 
    return (<br />);
  • JS reserved words are not allowed ("class" etc.) 
    return (<button className="btn-large"></button>);
  • Inline style can be written as:
    1. JS Object 
      return (<button style={this.btnStyle});
    2. Double curly brackets 
      return (<button style={{color: 'red'}});
